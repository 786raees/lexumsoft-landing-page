## Platform Notes for Nano Banana

Nano Banana is a newer/less-documented AI image generator. The following general strategies apply:

1. **Text rendering:** AI image generators commonly struggle with text. The prompt keeps text elements short and simple ("Your Local Plumbing Experts", "Call Now", "4.9 Rating"). If text garbles, reduce to just the headline and CTA button text. The hero headline and orange button are the two most important text elements -- everything else is secondary.

2. **UI/screenshot generation:** Generating realistic website screenshots is one of the harder tasks for AI image generators. The prompt uses several anchoring techniques:
   - Describes the website as displayed on a physical monitor (gives the AI a "frame" to work within)
   - Describes each section top-to-bottom in spatial order (helps the AI maintain layout coherence)
   - Specifies concrete design elements rather than abstract concepts ("three cards with rounded corners" not "a services section")
   - Uses a simple, common layout pattern (hero + trust bar + cards) that appears frequently in training data

3. **If Nano Banana has style/mode settings:** Use whichever mode is closest to "realistic" or "photography" for the desk/monitor framing. If it has a "design" or "graphic" mode, that may produce better UI rendering but worse environmental context.

4. **Aspect ratio:** If 1:1 is supported, use it for LinkedIn. If only standard ratios are available, 4:3 or 16:9 landscape will work -- 16:9 naturally suits a desktop monitor screenshot.

5. **Iteration strategy:** Generate 4 images per attempt. Evaluate primarily on:
   - Is the website layout recognizable as a real, professional website (not abstract art)?
   - Is the hero section clear with a visible headline and CTA button?
   - Are the service cards distinct and properly laid out?
   - Is text readable (even if not perfectly spelled)?

   If the layout is good but text is garbled, that image may still be usable -- viewers will register "professional website" from the layout, colors, and visual hierarchy even if individual words are not perfectly legible at LinkedIn scroll speed.

6. **Alternative approach (if screenshot quality is poor):** Instead of a website-on-a-monitor composition, try generating a flat website mockup without the monitor/desk context:

```
A flat UI design of a professional plumbing company website landing page, top-down view. Navy blue hero section at top with white heading "Your Local Plumbing Experts" and an orange call-to-action button. Below: a trust bar with gold stars and review count. Below that: three white service cards in a row with orange icons, titled "Drain Cleaning", "Water Heater Repair", "Emergency Plumbing." Light gray testimonials section at the bottom with gold star ratings. Clean modern sans-serif typography, navy blue and orange color palette, generous white space, pixel-perfect UI design. Behance-quality web design presentation.
```

This removes the monitor/desk environment and focuses the AI's rendering budget entirely on the website UI itself, which often produces cleaner results.

## Color Reference

| Element | Color | Reason |
|---------|-------|--------|
| Hero background | Deep navy blue to dark slate | Industry-standard trust color for plumbing/home services |
| CTA buttons | Bright orange | High contrast against navy, action-oriented, warm urgency |
| Body text | Dark navy / dark gray | Clean readability |
| Secondary backgrounds | White, very light gray | Clean, modern, premium feel |
| Trust/star ratings | Gold | Universal rating color |
| Service icons | Orange (matching CTA) | Consistent accent color throughout |

The navy + orange palette is deliberate. Navy conveys trust, reliability, and professionalism (critical for home services where a stranger enters your home). Orange conveys urgency and action (critical for conversion -- "call now"). This is the most effective color combination for plumbing/HVAC landing pages based on conversion data across the industry.
